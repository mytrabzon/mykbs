# MYKBS – Data Security Statement

**Last Updated: 26 February 2026**

MyKBS treats the protection of sensitive identity data as a **top priority.** This statement summarizes the technical and organizational security measures applied within the application.

---

## 1. Data Transmission

- All data transmission takes place over **encrypted connections** (TLS/SSL).
- Traffic between client and server is encrypted so that third parties cannot read the content.

---

## 2. Server and Access Control

- Data is stored on **secure server infrastructure**.
- Server access is restricted by **authorization and authentication** systems.
- Unauthorized access attempts are monitored and addressed.

---

## 3. Processing of Sensitive Data

- Sensitive fields such as identity and document numbers are **masked** in logs and reports (e.g. only last four digits visible).
- **Photographs of identity or passport documents are not stored or uploaded.** Only text data derived from the MRZ is processed.
- Sensitive data is **not permanently stored** on the user’s device.

---

## 4. Unauthorized Access and Breach Prevention

- **Technical and organizational measures** are in place to prevent unauthorized access and data breaches.
- Security incidents are assessed and users informed where appropriate.

---

## 5. No Commercial Use of Identity Data

**Identity data is not shared or sold for commercial purposes.** Data is processed only for identity verification, security, and legal compliance.

---

## 6. Updates

This statement may be updated to reflect changes in security standards or the application. The current text will be published in the app and on the website.

---

*This data security statement is effective as of 26 February 2026.*
